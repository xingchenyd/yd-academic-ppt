# Duration and scenario planner

## Default duration tiers

Use the user's explicit page count when given. Otherwise apply:

| Speaking time | Default total slides | Planning intent |
| --- | ---: | --- |
| 3 minutes | about 8 | Fast, high-information rhythm; no conventional agenda unless it adds real value |
| 5 minutes | about 12 | Complete compact story with room for proof/demo and a confident close |
| 10 minutes | about 20 | Full argument with supporting evidence, comparisons, limitations, or implementation detail |

Counts include cover and closing. Appendix slides do not count toward speaking slides. “About” allows a small adjustment for content quality, but do not silently turn 8 into 14 or 20 into 30.

If the user describes a long talk, gives a range, requests a detailed course, or supplies many mandatory sections, derive the page count from the prompt. Build a time ledger per slide and ensure the spoken script fits; do not extrapolate the three short tiers by a fixed formula.

## Scenario allocation

### Academic report / paper review

- 10–15% framing and research question.
- 20–30% theory, literature, or method.
- 35–45% evidence/results and interpretation.
- 15–25% synthesis, relationships, critique, limitations, and improvement.

### Thesis defense

- Minimize generic background.
- Make research questions, technical route, innovation, and results unmistakable.
- Reserve explicit space for contribution, limitations, and next work.
- Put backup derivations, full tables, and likely Q&A material in an appendix.

### Knowledge popularization

- Begin with a phenomenon, question, misconception, or relatable story.
- Explain through analogy, progressive reveal, worked example, and application.
- Use fewer citations on the visual surface while keeping source integrity.
- Favor friendly native typography and a small number of illustrative visuals.

### Course / training

- Organize around mental model → rule → example → variant → mistake → recap.
- Reuse “remember this” strips only when they help learning.
- Allow denser handout-style pages only when projection readability remains safe.

### Hackathon / startup / product pitch

- Open with a strong project promise or tension, not a table of contents.
- Show the product experience early.
- Prioritize demo, core loop, differentiation, evidence, implementation credibility, and impact/ask.
- Every slide should advance judge/investor belief rather than merely describe features.

### Game / interactive project

- Establish the world and player promise quickly.
- Explain the core loop visually.
- Show screenshots or faithful prototypes as evidence.
- Connect mechanics, progression, technology, social/educational value, and sustainability.

## Page-number system

- Short pitch: hide the cover number; use a discreet global `01 / 07` or current-page marker.
- Academic report/defense: use continuous global numbering; use `A-01` for appendix pages.
- Course series: use `module | 03 / 05` only when module-level navigation is genuinely useful.
- Do not copy the reference deck's module numbering into unrelated reports.
